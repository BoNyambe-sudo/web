# SEO Checklist for Blog Posts

Use this checklist when creating new blog posts to ensure optimal SEO performance.

## Pre-Writing Phase

- [ ] **Research Keywords**
  - Select 1-2 primary keywords from category keywords
  - Use Google Keyword Planner or Ubersuggest
  - Look for 10-100 monthly searches
  - Check competitor content for gaps

- [ ] **Plan Content Structure**
  - Main keyword in H1
  - 2-3 supporting keywords in H2s
  - Natural keyword placement (1-2% density)
  - Clear outline with logical flow

## Writing Phase

### Title Optimization (50-60 characters)
- [ ] Include primary keyword in title
- [ ] Use power words (Best, Complete, Ultimate, etc.)
- [ ] Make it compelling and clickable
- [ ] Keep it concise and descriptive

**Example**: "The Complete Guide to Web Development: From Planning to Launch"

### Meta Description (150-160 characters)
- [ ] Include primary keyword
- [ ] Summarize main benefit
- [ ] Include call-to-action
- [ ] Test character count

**Example**: "Master web development with our step-by-step guide covering planning, design, coding, testing, and deployment. Perfect for beginners and professionals."

### URL/Slug Optimization
- [ ] Include primary keyword
- [ ] Keep it short (3-5 words)
- [ ] Use hyphens to separate words
- [ ] Avoid numbers unless relevant

**Example**: `/blogs/complete-web-development-guide`

### Content Structure
- [ ] Single H1 tag (blog title only)
- [ ] Use H2s for main sections
- [ ] Use H3s for subsections
- [ ] Maintain proper hierarchy (no skipping levels)

### Word Count & Depth
- [ ] Minimum 1000 words for ranking potential
- [ ] Aim for 1500-2500 words for comprehensive coverage
- [ ] Write for humans first, SEO second
- [ ] Use short paragraphs (2-4 sentences max)

### Keyword Integration
- [ ] Primary keyword in first paragraph
- [ ] Keywords in H2 and H3 headings
- [ ] Natural language (avoid stuffing)
- [ ] Include keyword variations (synonyms)
- [ ] Target 1-2% keyword density

**Natural Integration Example**:
Instead of: "Web development is web development for beginners"
Write: "Web development skills are essential for beginners entering the tech field"

### Images & Media
- [ ] At least one image per 500 words
- [ ] Descriptive alt text (5-120 characters)
- [ ] Relevant captions where appropriate
- [ ] Optimized file sizes (<100KB for web)
- [ ] Proper aspect ratios for responsive design

**Alt Text Example**:
❌ "image.jpg" or "pic1"
✅ "HTML and CSS code editor displaying responsive web design layout"

### Internal Linking
- [ ] Link to 2-3 related blog posts
- [ ] Use descriptive anchor text
- [ ] Link to pillar content and topic clusters
- [ ] Ensure linked pages are relevant
- [ ] Use first link on page for most important target

**Internal Link Example**:
```
Learn more about [mobile app development strategies](link) to expand your web presence.
```

### External Links
- [ ] Link to 1-2 authoritative sources
- [ ] Use rel="noopener noreferrer" for external links
- [ ] Link to reputable, relevant sources only
- [ ] Avoid linking to competitors' best content

## Metadata Setup

### Blog Post Metadata (Content Management)
- [ ] **Title**: Keyword-rich, 50-60 characters
- [ ] **Slug**: Keyword-focused URL
- [ ] **Meta Description**: Compelling, 150-160 characters
- [ ] **Category**: Select from 18 available categories
- [ ] **Tags**: 3-5 relevant tags (not overly specific)
- [ ] **Featured Image**: High-quality, optimized image
- [ ] **Author**: Correct author information
- [ ] **Publication Date**: Accurate publish date
- [ ] **Canonical URL**: Default (your site URL)

### Tags Best Practices
- [ ] Max 5 tags per post
- [ ] Avoid single-use tags
- [ ] Use 2-3 main topic tags
- [ ] Add 1-2 specific technique tags
- [ ] Maintain consistent tag naming

### Category Selection
Choose the primary category from:
Technology, Lifestyle, Health, Photography, Sports, Business, Religious, Politics, Science, Art, Music, Film & TV, Fashion, Food, Travel, Finance, Education, Entertainment

## Technical SEO

### Structured Data
- [ ] Blog post schema will auto-generate from metadata
- [ ] Verify schema at https://schema.org/validate/
- [ ] Check author and publication date accuracy
- [ ] Ensure image is included in schema

### Mobile Optimization
- [ ] Test on mobile devices
- [ ] Check responsive design
- [ ] Test touch elements (links, buttons)
- [ ] Verify readability on small screens
- [ ] Check Core Web Vitals

### Performance
- [ ] Minimize JavaScript in post content
- [ ] Lazy load images
- [ ] Use CDN for images
- [ ] Minify CSS and JavaScript

## Content Quality Checklist

### Readability
- [ ] Easy-to-read headings
- [ ] Short sentences (15-20 words avg)
- [ ] Short paragraphs (3-5 lines max)
- [ ] Bullet points and lists
- [ ] Clear transitions between topics

### Accuracy & Authority
- [ ] Fact-check all claims
- [ ] Include data sources and studies
- [ ] Update with latest information
- [ ] Review by subject matter expert
- [ ] Provide actionable insights

### Engagement
- [ ] Opening hook in first paragraph
- [ ] Answer "Why should I read this?" early
- [ ] Include relevant examples
- [ ] Tell stories when appropriate
- [ ] End with clear takeaway

### Formatting
- [ ] Proper spacing and formatting
- [ ] Code blocks with syntax highlighting
- [ ] Quotes formatted distinctly
- [ ] Lists with consistent formatting
- [ ] Tables for data comparison

## Pre-Publication

### Final Review
- [ ] Proofread for grammar/spelling
- [ ] Check all links (internal and external)
- [ ] Verify images load correctly
- [ ] Test code examples if included
- [ ] Validate HTML structure

### SEO Audit
- [ ] Run Lighthouse audit
- [ ] Check Google PageSpeed score
- [ ] Validate structured data
- [ ] Test mobile responsiveness
- [ ] Verify Core Web Vitals

### Preview
- [ ] Preview on desktop
- [ ] Preview on tablet
- [ ] Preview on mobile
- [ ] Check dark mode appearance
- [ ] Test with accessibility tools

## Post-Publication

### Immediate (Day 1)
- [ ] Verify publish date is correct
- [ ] Test all links work
- [ ] Check page renders correctly
- [ ] Share on social media with custom description
- [ ] Monitor for any errors in console

### Week 1
- [ ] Submit to Google Search Console
- [ ] Monitor for indexing issues
- [ ] Check initial traffic
- [ ] Respond to comments
- [ ] Update related blog posts with links if relevant

### Ongoing
- [ ] Monitor search rankings for target keywords
- [ ] Track organic traffic
- [ ] Monitor user engagement (time on page, bounce rate)
- [ ] Update outdated information quarterly
- [ ] Build backlinks through promotion

## SEO Quick Reference

### Keyword Density Target
- Primary keyword: 0.5-1.5% (2-4 times in 1000 words)
- Secondary keywords: 0.5-1% (similar distribution)
- Keyword variations: Naturally throughout

### Optimal Content Metrics
- Word count: 1000-2500 words
- Paragraphs: 3-5 sentences each
- Sentences: 15-20 words average
- Headings: One every 200-300 words
- Images: One per 500 words

### Internal Link Targets
- 2-3 links per post
- Relevant and contextual
- Descriptive anchor text
- Mix of different target pages
